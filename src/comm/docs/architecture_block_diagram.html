Relative localization

platoon_localization/relative_localization.launch.py wraps relative_localization_eskf.

  • ArUco board pose correction
  • IMU yaw propagation and bias handling
  • LiDAR scan input enabled by default
  • GPS relative update when both odom candidates are fresh

Map Anchor Decision

Target TF chain

map -> leader/base_link -> follower/base_link -> follower/follower_gps

Follower GPS does not own map -> follower/base_link. It is an odometry measurement that can improve either the relative ESKF or future leader map anchor.

Future fallback logic

If leader RTK is poor but follower RTK is fixed and relative pose is healthy:

map->leader = map->follower_candidate * inverse(leader->follower)

The published TF edge is still only map -> leader/base_link.
