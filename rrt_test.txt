GP_Decision Complex RRT Home Test Strategy
==========================================

Goal
----
- 집에서는 실차 주행이 아니라 ROS2 토픽 흐름과 planner 출력 검증을 목표로 한다.
- Arduino, 모터, serial bridge는 처음에는 연결하지 않는다.
- 확인 순서:
  1. 빌드 가능 여부
  2. launch 후 노드/토픽 생성 여부
  3. Highway command 흐름
  4. Complex RRT local path 생성
  5. Complex command 선택
  6. Safety stop에서 throttle 차단


0. Safety Rule
--------------
- 집 테스트에서는 반드시 serial bridge를 끈다.

  ros2 launch auto_drive bringup_single_f9p.launch.py \
    csv_file_path:=/path/to/test.csv \
    use_serial_bridge:=false \
    use_rviz:=true

- Arduino/모터 연결 전까지는 아래 토픽만 확인한다.
  - /auto_steer_angle
  - /throttle_from_planning
  - /throttle_cmd
  - /complex/local_path
  - /planning_command_source

5. Complex RRT Test Without GPS
-------------------------------
Use this when /complex/rrt_target is not generated because GPS/TF is unavailable.

Terminal A: publish target manually.

  ros2 topic pub /complex/rrt_target geometry_msgs/msg/PointStamped "{
    header: {frame_id: 'vehicle_ref'},
    point: {x: 6.0, y: 0.0, z: 0.0}
  }" -r 10

Terminal B: publish cone map manually.

  ros2 topic pub /complex/cones geometry_msgs/msg/PoseArray "{
    header: {frame_id: 'vehicle_ref'},
    poses: [
      {position: {x: 2.0, y: 1.0, z: 0.0}, orientation: {w: 1.0}},
      {position: {x: 3.5, y: 1.1, z: 0.0}, orientation: {w: 1.0}},
      {position: {x: 5.0, y: 1.0, z: 0.0}, orientation: {w: 1.0}},
      {position: {x: 2.0, y: -1.0, z: 0.0}, orientation: {w: 1.0}},
      {position: {x: 3.5, y: -1.1, z: 0.0}, orientation: {w: 1.0}},
      {position: {x: 5.0, y: -1.0, z: 0.0}, orientation: {w: 1.0}}
    ]
  }" -r 10

Check:

  ros2 topic echo /complex/rrt_status
  ros2 topic echo /complex/local_path
  ros2 topic echo /complex/auto_steer_angle
  ros2 topic echo /complex/throttle_from_planning

Expected:
- /complex/rrt_status = direct_clear, reached_target, or best_branch
- /complex/local_path.poses is not empty
- /complex/auto_steer_angle publishes values
- /complex/throttle_from_planning publishes values

7. Safety Stop Test
-------------------
Set manual stop:

  ros2 topic pub /manual_stop std_msgs/msg/Bool "{data: true}" --once

Check:

  ros2 topic echo /safety_active
  ros2 topic echo /safety_status
  ros2 topic echo /throttle_cmd

Expected:
- /safety_active = true
- /safety_status = STOP_MANUAL
- /throttle_cmd = 0.0

Release:

  ros2 topic pub /manual_stop std_msgs/msg/Bool "{data: false}" --once

Expected:
- safety hysteresis 이후 /safety_active = false
- /throttle_cmd resumes according to mission policy

8-1. Velodyne TF Check
----------------------
If /detected_objects marker.header.frame_id is velodyne, RRT needs this TF:

  vehicle_ref -> velodyne

Default launch publishes an identity static TF. Override it with measured LiDAR
mounting values when the LiDAR origin is not the same as vehicle_ref:

  ros2 launch auto_drive bringup_single_f9p.launch.py \
    csv_file_path:=/path/to/path.csv \
    velodyne_x:=0.0 velodyne_y:=0.0 velodyne_z:=0.0 \
    velodyne_yaw:=0.0 velodyne_pitch:=0.0 velodyne_roll:=0.0

If another LiDAR launch already publishes vehicle_ref -> velodyne:

  ros2 launch auto_drive bringup_single_f9p.launch.py \
    csv_file_path:=/path/to/path.csv \
    publish_velodyne_tf:=false

Check:

  ros2 run tf2_ros tf2_echo vehicle_ref velodyne

Pass condition:
- tf2_echo prints a transform without lookup/extrapolation errors.
- /complex/rrt_obstacles appears at the same x/y position as cone objects.


9. Common Failure Cases
-----------------------
Case: /complex/rrt_status = missing_target
- /complex/rrt_target is not being published.
- Check /csv_path and TF csv -> vehicle_ref.
- For home test, manually publish /complex/rrt_target.

Case: /complex/rrt_status = missing_cones
- No cone input.
- Publish /complex/cones manually.
- Or connect LiDAR perception to /detected_objects.
- For /detected_objects MarkerArray, check that each cone has:
  - ns=objects, id=N, pose.position as center
  - ns=info, id=10000+N, text first line as Cone
  - object frame such as velodyne must be TF-transformable to vehicle_ref

Case: /planning_command_source = highway while expecting complex
- Check /drive_context.
- Check /mission_state.
- Command mux uses /mission_state and /drive_context.

Case: /auto_steer_angle is zero
- Selected planner command may be stale.
- Check /planning_command_source for highway_stale or complex_stale.
- Check selected planner output topic frequency.

Case: /throttle_cmd is zero
- mission_supervisor may be safety active.
- Check /safety_active and /safety_status.
- Check /manual_stop, /safety_stop, /traffic_stop, /intersection, /roi_warning.
